# Контракт движка новеллы (Cutie)

Точный формат контента. Не пиши сцены по памяти — сверяйся отсюда. Источники:
`src/engine/types.ts`, `src/engine/condition.ts`, `src/store/gameStore.ts`,
`src/content/scenes/index.ts`, `src/content/manifest.json`.

## Где живёт контент

- Сцены — по одному JSON-файлу на сцену в `src/content/scenes/`.
- Каждая сцена регистрируется в `src/content/scenes/index.ts`: импорт + строка в объекте
  `scenes`. **Файл сам по себе движок не видит — без строки в реестре сцены нет.**
- Стартовая сцена задаётся константой `START_SCENE_ID` в том же `index.ts` (сейчас `'start'`).
- Спрайты и аудио декларируются в `src/content/manifest.json`.

## Формат сцены (`Scene`)

```jsonc
{
  "id": "scene_id",              // обязателен, должен совпадать с ключом в index.ts
  "music": "bgm_theme",          // опц. ключ из manifest.audio; меняет текущий трек
  "sprites": [                   // опц. набор спрайтов на экране (полностью заменяет предыдущий)
    { "key": "cinnamoroll_happy", "position": "left|center|right", "expression": "..." }
  ],
  "sfx": "sfx_chime",            // опц. одноразовый звук при входе в сцену
  "text": [                      // обязателен: массив реплик, показываются по очереди
    { "speaker": "Имя", "body": "Реплика." },   // speaker опц. — без него это нарратив
    { "body": "Строка нарратива без говорящего." }
  ],
  "setFlags": { "metCat": true },// опц. флаги, выставляемые при ВХОДЕ в сцену
  "choices": [ /* см. ниже */ ], // опц. выборы игрока
  "next": "other_scene",         // опц. переход без выбора (линейный)
  "isEnding": true               // опц. помечает концовку (нет next/choices)
}
```

Правила ветвления в одной сцене:
- **Не-концовка** обязана иметь `choices` ИЛИ `next`. Иначе игрок в тупике.
- **Концовка**: `isEnding: true`, без `next` и `choices`.
- Если есть и `choices`, и `next` — выборы имеют приоритет в UI; не смешивай без нужды.

## Формат выбора (`Choice`)

```jsonc
{
  "label": "Текст кнопки для игрока",
  "condition": "affection >= 3 && metCat == true",  // опц.; выбор скрыт, если ложно
  "setFlags": { "affection": 2 },                    // опц.; применяется при клике
  "next": "target_scene"                             // обязателен: id следующей сцены
}
```

## Флаги — КРИТИЧНО: это присваивание, не инкремент

`setFlags` мёржится через spread (`{ ...flags, ...newFlags }`). Значение **перезаписывается
абсолютно**. `"affection": 1` ставит 1, а не +1.

- Инкремента нет. Чтобы "накопить симпатию", проектируй пороговые значения заранее и
  выставляй нужное абсолютное число на каждой ветке (в референсном контенте так: 0 → 1 → 2).
- Планируй числовую шкалу флага в `FLAGS.md`: какие значения на каких развилках, какой
  порог гейтит какую концовку.
- Тип флага: `number | boolean | string`.
- Порядок применения при переходе по выбору: сперва `choice.setFlags`, затем вход в
  целевую сцену применяет её `scene.setFlags`. Оба — абсолютная запись.

## Условия (`condition`)

Парсер безопасный (без `eval`), поддерживает **только**:
- Сравнения: `>= <= == != > <` (например `affection >= 3`, `route == "cat"`).
- Соединение через `&&`. **`||` и скобок НЕТ.** Нужна дизъюнкция — сделай два отдельных
  выбора с разными условиями, ведущих в одну сцену.
- Голое имя флага (`metCat`) — истинно, если значение truthy.
- Отсутствующий флаг в сравнении трактуется как `0`.

Литералы: `true`/`false` → boolean, число → number, иначе строка (кавычки снимаются).

## Спрайты и аудио (`manifest.json`)

Любой `key` в `sprites`/`music`/`sfx` должен существовать в `manifest.json`:

```jsonc
{
  "sprites": {
    "cinnamoroll_happy": { "path": "", "description": "счастливый, машет лапкой" }
  },
  "audio": {
    "bgm_theme": { "path": "", "description": "лёгкая зацикленная мелодия" }
  }
}
```

- Пустой `path` = рендерится плейсхолдер по `description`. Это норма — можно писать всю
  новеллу на плейсхолдерах, арт подставить позже.
- `sprites` в сцене **полностью заменяют** предыдущий набор (не добавляют). Хочешь оставить
  персонажа на экране — перечисли его снова.

## Состояние, история, сейвы (уже в движке — писать не надо)

- Флаги накапливаются в сторе и переносятся между сценами.
- `text` каждой посещённой сцены попадает в историю (лог) автоматически.
- Сейв/загрузка/удаление слотов и History-модал уже реализованы — сохраняют `sceneId +
  flags + history`. Твоя задача — только контент; UI-механику не трогай.

## Ловушки, которые роняют движок или ломают игру

1. **Битый `next`/`choice.next`** на несуществующий id → `scenes[id]` undefined →
   `enterScene(undefined)` падает. Каждый переход должен ссылаться на зарегистрированную сцену.
2. **Сцена-файл без строки в `index.ts`** — не существует для движка.
3. **Осиротевшая сцена** — есть в реестре, но никто на неё не ведёт: недостижима.
4. **Не-концовка без `next` и `choices`** — тупик, дальше пройти нельзя.
5. **`condition` на флаг, который нигде раньше не выставлен** — всегда `0`/ложь: выбор
   или недостижим, или ведёт не туда, куда ждёшь.
6. **Концовка, заблокированная условиями навсегда** — ни один путь не даёт нужный набор
   флагов. Проверяется трассировкой графа.
7. **Использование `||` или скобок в `condition`** — не поддерживается, молча сломается.
