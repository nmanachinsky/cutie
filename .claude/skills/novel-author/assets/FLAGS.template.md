# Реестр флагов: <НАЗВАНИЕ>

Помни: setFlags — АБСОЛЮТНОЕ присваивание, не инкремент. Планируй значения на развилках.
condition поддерживает только сравнения и `&&` (нет `||`, нет скобок).

## Флаги
| Флаг | Тип | Назначение | Значения по развилкам (где выставляется) | Где читается (condition) |
|------|-----|-----------|------------------------------------------|--------------------------|
| affection | number | шкала симпатии | meet:choice — walk даёт 1, quiet 0; walk:choice — 2/1 | hub: >=4 best, >=2 good |
| route | string | выбранная линия | <scene:значение> | <scene: route == "…"> |
| foundKey | boolean | найден ключ | <scene: true> | <scene: foundKey == true> |

## Пороги концовок
- ending_best: affection >= 4
- ending_good: affection >= 2
- ending_calm: иначе (fallback)

## Отложенные последствия
<Флаг X, выставленный в сцене A, читается в сцене B через N шагов — чтобы ранний выбор ощущался.>

## Инвариант
Каждый флаг в этой таблице обязан:
- выставляться хотя бы в одной сцене/выборе ДО точки чтения;
- читаться хотя бы одним condition или разводить концовки.
Флаг, который только пишется но нигде не решает — удали (YAGNI).
